# 编辑器自动面检测功能实现

## 概述

根据 `facing-explanation.md` 的说明，编辑器现在支持自动检测玩家视线指向的判定面，无需手动切换。

## 主要变更

### 1. 新增 EditorFaceDetector 类

**功能**：自动检测玩家视线指向哪个判定面

**判定面定义**：
- Z+ 方向 (z=4) → W 面 (前)
- X+ 方向 (x=4) → A 面 (左)
- Z- 方向 (z=-4) → S 面 (后)
- X- 方向 (x=-4) → D 面 (右)

**判定面范围**：
- W/S 面: x ∈ [-3, 3], y ∈ [-3, 3]
- A/D 面: z ∈ [-3, 3], y ∈ [-3, 3]

**检测逻辑**：
1. 计算视线与各个判定面的交点
2. 找到最近的面（且在玩家前方）
3. 验证交点是否在判定面范围内
4. 返回检测到的面，如果没有则返回 null

### 2. 更新 EditorPreviewCursor

**变更**：
- 使用 `EditorFaceDetector.detectFacingFace()` 自动检测目标面
- 预览光标自动显示在玩家视线指向的面上
- 发光颜色自动匹配检测到的面
- 如果没有指向任何面，不显示预览光标

**好处**：
- 无需手动切换面，更直观
- 预览光标始终显示在正确的位置
- 发光颜色准确反映目标面

### 3. 更新 Action Bar 显示

**变更**：
- 显示自动检测到的当前面
- 如果没有指向任何面，显示 "无"

### 4. 保留的功能

**面切换功能**：
- 虽然现在支持自动检测，但保留了手动切换面的功能（通过滚轮槽位 9）
- 自动检测的面会更新 session 的 currentFace
- 这样在放置音符时使用的是自动检测的面

## 使用体验

### 之前的工作流程：
1. 按 9 键切换到目标面
2. 移动视角找到位置
3. 右键放置音符

### 现在的工作流程：
1. 直接看向目标面的位置
2. 预览光标自动出现在正确的面上
3. 右键放置音符

## 技术细节

### 面检测算法

```java
// 1. 计算视线与平面的交点
// 对于 Z 平面 (W/S): t = (targetZ - playerZ) / directionZ
// 对于 X 平面 (A/D): t = (targetX - playerX) / directionX

// 2. 计算交点坐标
intersection = playerPos + direction * t

// 3. 检查交点是否在面范围内
// W/S: x ∈ [-3, 3], y ∈ [-3, 3]
// A/D: z ∈ [-3, 3], y ∈ [-3, 3]

// 4. 选择最近的有效面
```

## 已知限制

1. **面检测精度**：当玩家视线接近两个面的边界时，可能会在两个面之间快速切换
2. **范围限制**：只有当视线交点在判定面范围内时才会显示预览
3. **平行情况**：当视线平行于判定面时，不会检测到该面

## 后续优化建议

1. **添加面切换平滑过渡**：避免在边界处频繁切换
2. **添加面指示器**：在 HUD 中显示当前指向的面
3. **优化边界检测**：在边界附近增加一定的容差
4. **添加面锁定功能**：允许玩家临时锁定某个面，避免意外切换
