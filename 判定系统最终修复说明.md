## 修复的三个问题

### 1. ✅ FLICK 音符 NPE 保护

**问题:** FLICK 音符没有 Interaction 实体，`calculateTimingOffset` 方法会导致 NullPointerException

**修复位置:** `JudgmentManager.java:25-31`

**修复方案:**
```java
public int calculateTimingOffset(NoteEntity noteEntity, double speed) {
    // FLICK 音符没有 Interaction，使用 BlockDisplay 的位置
    Location notePos;
    if (noteEntity.getInteraction() != null) {
        notePos = noteEntity.getInteraction().getLocation();
    } else if (noteEntity.getBlockDisplay() != null) {
        notePos = noteEntity.getBlockDisplay().getLocation();
    } else {
        return 0;  // 不应该发生
    }

    Face face = noteEntity.getFace();
    double distance = CoordinateSystem.calculateDistance(notePos, face, centerX, centerZ);
    return (int) ((distance / speed) * 50);
}
```

**说明:**
- 优先使用 Interaction 的位置（TAP, DOUBLE, DRAG, HOLD）
- 如果没有 Interaction，使用 BlockDisplay 的位置（FLICK）
- 避免了 NullPointerException

### 3. ✅ 修正 FLICK 判定方法

**问题:** FLICK 判定范围不正确，应该检查玩家视角是否在目标方向的 45°-135° 范围内

**修复位置:** `GameSession.java:284-313`

**原实现（错误）:**
```java
// 计算目标方向的 yaw
float targetYaw = normalizeAngle(baseYaw + 90f);  // 或 -90f

// 检查是否在 ±45° 范围内
float angleDiff = Math.abs(normalizeAngle(playerYaw - targetYaw));
return angleDiff <= 45f;  // 错误：这是检查 0°-45°
```

**新实现（正确）:**
```java
// 计算玩家相对于基准方向的角度偏移
float angleFromBase = normalizeAngle(playerYaw - baseYaw);

// 检查是否在目标方向的 45°-135° 范围内
if (targetDirection.equalsIgnoreCase("left")) {
    // 左转：角度应该在 45° 到 135° 之间（正值）
    return angleFromBase >= 45f && angleFromBase <= 135f;
} else {
    // 右转：角度应该在 -135° 到 -45° 之间（负值）
    return angleFromBase <= -45f && angleFromBase >= -135f;
}
```

**判定范围说明:**

```
示例：前方面板 (W, baseYaw=0°) 向左转

玩家 yaw    angleFromBase    判定结果
---------   -------------    --------
0°          0°               MISS (不在 45°-135° 范围)
45°         45°              EXACT (刚好在范围边界)
90°         90°              EXACT (在范围中心)
135°        135°             EXACT (刚好在范围边界)
180°        180°             MISS (超出范围)
```

**角度范围可视化:**

```
        -90° (D面)
           |
           |
180° ------+------ 0° (W面)
(S面)      |
           |
        90° (A面)

W面向左FLICK (turn="left"):
- 基准: 0°
- 判定范围: 45° ~ 135° (90° 扇形区域)
- 中心: 90° (正左方)

W面向右FLICK (turn="right"):
- 基准: 0°
- 判定范围: -135° ~ -45° (90° 扇形区域)
- 中心: -90° (正右方)
```
