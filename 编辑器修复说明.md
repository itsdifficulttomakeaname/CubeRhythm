# 编辑器修复说明

## 修复内容

### 1. 谱面文件路径修复

**问题**：谱面文件直接保存在 `plugins/CubeRhythm/` 目录下

**修复**：
- 修改 `EditorFileUtil.CHARTS_DIR` 为 `plugins/CubeRhythm/charts/`
- 在保存时自动创建 `charts` 文件夹（如果不存在）

**代码位置**：
```java
// EditorFileUtil.java
private static final File CHARTS_DIR = new File(Main.instance.getDataFolder(), "charts");

// saveSession 方法中
if (!CHARTS_DIR.exists()) {
    CHARTS_DIR.mkdirs();
}
```

### 2. 鼠标滚轮行为修复

**问题**：原实现的滚轮行为不正确

**修复后的行为**：

| 滚动到槽位 | 功能 | 说明 |
|-----------|------|------|
| 1 (索引0) | 切换音符类型 | TAP → DOUBLE → DRAG → HOLD → FLICK(左) → FLICK(右) → TAP |
| 2-4 (索引1-3) | 后退一拍 | 向前滚动鼠标轮 |
| 5 (索引4) | 魔杖槽位 | 操作完成后自动返回此槽位 |
| 6-8 (索引5-7) | 前进一拍 | 向后滚动鼠标轮 |
| 9 (索引8) | 切换判定面 | W → A → S → D → W |

**关键改动**：

1. **按拍移动而非按步移动**
   - 添加 `EditorSession.addBeat(int)` 方法
   - 每次移动一整拍（`tick += amount * stepLength`）

2. **槽位检测逻辑**
   ```java
   if (newSlot >= 2 && newSlot <= 4) {
       // 后退一拍
   } else if (newSlot >= 6 && newSlot <= 8) {
       // 前进一拍
   } else if (newSlot == 1) {
       // 切换音符类型
   } else if (newSlot == 0) {
       // 切换判定面
   }
   ```

3. **自动返回槽位5**
   ```java
   // 操作完成后，切回槽位 4 (索引从0开始，所以是4)
   player.getInventory().setHeldItemSlot(4);
   ```

## 使用说明

### 鼠标滚轮操作

1. **切换音符类型**
   - 向上滚动到槽位1
   - 音符类型会循环切换
   - 自动返回魔杖槽位

2. **切换判定面**
   - 向上滚动到槽位9
   - 判定面会循环切换
   - 自动返回魔杖槽位

3. **时间导航**
   - 向上滚动（到槽位2-4）：后退一拍
   - 向下滚动（到槽位6-8）：前进一拍
   - 自动返回魔杖槽位

### 示例操作流程

```
初始状态：持有魔杖（槽位5）

1. 向上滚动一格 → 到达槽位6 → 前进一拍 → 自动返回槽位5
2. 向上滚动两格 → 到达槽位7 → 前进一拍 → 自动返回槽位5
3. 向上滚动四格 → 到达槽位9 → 切换判定面 → 自动返回槽位5
4. 向下滚动一格 → 到达槽位4 → 后退一拍 → 自动返回槽位5
```

## 技术细节

### EditorSession 新增方法

```java
/**
 * 前进/后退指定拍数
 */
public void addBeat(int amount) {
    tick += amount * stepLength;
    if (tick < 0) {
        tick = 0;
    }
}
```

### EditorListener 修改

- 移除了 `isScrollUp()` 方法（不再需要判断滚动方向）
- 直接根据目标槽位判断操作类型
- 所有操作完成后统一返回槽位4

### 文件路径处理

```java
// 确保 charts 文件夹存在
if (!CHARTS_DIR.exists()) {
    CHARTS_DIR.mkdirs();
}

// 构建文件路径
File file = new File(CHARTS_DIR, session.getChartId() + ".json");
```

## 兼容性说明

### 向后兼容

- 如果旧谱面文件在 `plugins/CubeRhythm/` 目录下，需要手动移动到 `charts/` 文件夹
- 或者使用命令重新加载：`/editor load <chartId>` 会在 `charts/` 文件夹中查找

### 迁移步骤

1. 创建 `plugins/CubeRhythm/charts/` 文件夹
2. 将所有 `.json` 谱面文件移动到 `charts/` 文件夹
3. 重启服务器或重新加载插件
