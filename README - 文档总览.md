## 核心概念速查

### 音符类型

| 类型 | 说明 | 颜色 | 操作 |
|------|------|------|------|
| Tap | 点击音符 | 浅蓝色 | 单击鼠标 |
| Hold | 长按音符 | 白色 | 按住右键 |
| Drag | 拖动音符 | 黄色 | 对准（无需按键） |
| Flick | 滑动音符 | 品红/红色 | 旋转90° |
| Double | 双音符 | 橙色 | 双击左右键 |
| Execution | 触发器 | 黑色 | 自动执行 |

### 判定面

| 代码 | 方向 | 颜色标识 | 旋转角度 |
|------|------|---------|---------|
| w | 前 | 白色 | 0° |
| a | 左 | 黄色 | 90° |
| s | 后 | 橙色 | 180° |
| d | 右 | 红色 | 270° |

### 关键变量

#### 游戏模式
```
{loadedNotes::type::%id%} - 音符类型
{loadedNotes::time::%id%} - 音符时间
{loadedNotes::face::%id%} - 判定面
{loadedNotes::x::%id%} - X坐标
{loadedNotes::y::%id%} - Y坐标
```

#### 编辑器模式
```
{editorSave::type::%uuid%} - 音符类型
{editorSave::time::%uuid%} - 时间（秒）
{editorSave::loc::%uuid%} - 位置
{editorSave::face::%uuid%} - 判定面
{editorSave::glowing::%uuid%} - 发光
{editorSave::tag::%uuid%} - 标签
```

### 关键公式

#### 时间计算
```
音符时间 = tick / stepLength * (60 / bpm) + preTime
```

#### 距离计算
```
Z坐标 = speed * 20 * (noteTime - currentTime) + 4
```

#### 生成条件
```
距离 < 50 且 实体数 < 100
```

## 数据流程图

```
服务器启动
    ↓
加载所有 *_properties.sk
    ↓
注册谱面信息到 {loadedCharts::*}
    ↓
玩家执行 /play
    ↓
显示谱面列表 GUI
    ↓
玩家选择谱面
    ↓
禁用其他谱面脚本
    ↓
启用选中谱面脚本（触发 on load）
    ↓
loadChart() 调用音符函数
    ↓
音符注册到 {loadedNotes::*}
    ↓
玩家执行 /start（或自动触发）
    ↓
异步排序音符（按时间升序）
    ↓
重组为顺序索引
    ↓
应用时间偏移（+3s + 玩家偏移 + 谱面偏移）
    ↓
显示歌曲信息
    ↓
播放音频
    ↓
启动渲染循环（every ticks）
    ↓
每Tick检查需要生成的音符
    ↓
调用 drawXXX() 函数生成实体
    ↓
音符移动到判定面
    ↓
检测玩家输入
    ↓
计算判定结果
    ↓
更新分数和连击
    ↓
删除已判定音符
    ↓
所有音符处理完毕
    ↓
游戏结束
```

## JSON 格式核心结构

### 完整谱面 JSON
```json
{
  "version": "1.0.0",
  "metadata": {
    "id": "chart_id",
    "title": "曲名",
    "artist": "作曲家",
    "charter": "谱师",
    "difficulty": {...},
    "audio": {...},
    "timing": {...}
  },
  "events": [...],
  "notes": [...]
}
```

### 音符 JSON（Tap示例）
```json
{
  "type": "tap",
  "time": 14.77,
  "face": "w",
  "position": {"x": 1, "y": 0},
  "glowing": false,
  "tag": ""
}
```

### 渲染配置 JSON
```json
{
  "renderConfig": {
    "globalSettings": {
      "tickRate": 20,
      "maxConcurrentNotes": 100,
      "spawnDistance": 50
    },
    "noteTypes": {...}
  }
}
```

## 常见问题解答

### Q: 为什么要转换为 JSON 格式？
A: JSON 格式具有以下优势：
- 标准化数据结构
- 易于验证和解析
- 支持版本控制
- 便于跨平台使用
- 可压缩和流式加载

### Q: 转换后会影响现有谱面吗？
A: 建议采用双格式并存方案：
- 继续支持 .sk 格式
- 新谱面使用 JSON 格式
- 提供转换工具
- 逐步迁移

### Q: 如何将现有谱面转换为 JSON？
A: 参考 [铺面文件格式说明.md](#1-铺面文件格式说明) 的"格式转换示例"章节，或使用转换工具。

### Q: 编辑器生成的代码可以直接使用吗？
A: 可以，使用 `/gen` 命令生成的代码可以直接复制到谱面文件的 `loadChart()` 函数中。

### Q: 如何优化大型谱面的加载速度？
A: 参考 [铺面加载流程说明.md](#3-铺面加载流程说明) 的"未来JSON格式改进建议"章节。

## 技术规格总结

### 文件格式
- **当前格式**: Skript (.sk)
- **建议格式**: JSON (.json)
- **编码**: UTF-8
- **Schema版本**: 1.0.0

### 性能指标
- **渲染频率**: 20 Ticks/秒
- **最大实体数**: 100个交互实体
- **生成距离**: < 50格
- **可视距离**: -1 到 50格

### 数据规模
- **小型谱面**: 100音符，10-20 KB
- **中型谱面**: 500音符，50-100 KB
- **大型谱面**: 2000音符，200-400 KB

### 加载时间
- **小型谱面**: ~5.5秒
- **中型谱面**: ~6.0秒
- **大型谱面**: ~7.0秒
