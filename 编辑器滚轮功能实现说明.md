# 编辑器鼠标滚轮功能实现说明

## 功能概述

实现了与 Skript 版本相同的鼠标滚轮交互功能，允许玩家通过滚动鼠标轮来：
- 切换音符类型
- 切换判定面
- 在时间轴上前进/后退

## 实现原理

### 事件检测

使用 `PlayerItemHeldEvent` 来检测鼠标滚轮滚动：
- 当玩家滚动鼠标轮时，Minecraft 会尝试切换物品栏槽位
- 通过检测槽位变化方向来判断滚轮方向
- 取消事件以防止实际切换槽位

### 滚动方向判断

```java
boolean scrollUp = isScrollUp(previousSlot, newSlot);
```

- 向上滚动：`newSlot > previousSlot`（除了 8→0 的循环情况）
- 向下滚动：`newSlot < previousSlot`（除了 0→8 的循环情况）

### 功能分配

根据玩家当前持有的槽位执行不同操作：

| 槽位 | 功能 | 说明 |
|------|------|------|
| 0 | 切换音符类型 | TAP → DOUBLE → DRAG → HOLD → FLICK(左) → FLICK(右) → TAP |
| 8 | 切换判定面 | W(前) → A(左) → S(后) → D(右) → W |
| 其他 | 时间导航 | 向上滚动前进一格，向下滚动后退一格 |

## 核心类

### 1. EditorSession

添加了以下方法：

```java
// 前进/后退指定步数
public void addTick(int amount)

// 循环切换音符类型
public void cycleNoteType()

// 循环切换判定面
public void cycleFace()
```

添加了字段：
```java
private String flickDirection;  // "left" or "right" for FLICK notes
```

### 2. EditorListener

新建的事件监听器类，处理所有编辑器交互：

**主要方法：**
- `onItemHeldChange()` - 处理鼠标滚轮事件
- `handleNoteTypeCycle()` - 处理音符类型切换
- `handleFaceCycle()` - 处理判定面切换
- `handleTimeNavigation()` - 处理时间导航
- `updateNoteTypeItem()` - 更新物品栏显示
- `updateFaceItem()` - 更新判定面物品显示

## 音符类型循环

### 循环顺序

1. **TAP** (浅蓝色羊毛) - §bTap
2. **DOUBLE** (橙色羊毛) - §6Double
3. **DRAG** (黄色羊毛) - §eDrag
4. **HOLD** (白色羊毛) - §fHold
5. **FLICK(左)** (品红色羊毛) - §a← §dFlick
6. **FLICK(右)** (红色羊毛) - §cFlick §a→
7. 回到 **TAP**

### 物品显示

每种音符类型对应不同颜色的羊毛：
- TAP: `LIGHT_BLUE_WOOL`
- DOUBLE: `ORANGE_WOOL`
- DRAG: `YELLOW_WOOL`
- HOLD: `WHITE_WOOL`
- FLICK(左): `MAGENTA_WOOL`
- FLICK(右): `RED_WOOL`

## 判定面循环

### 循环顺序

1. **W - 前** (白色染色玻璃) - §f前 (W)
2. **A - 左** (黄色染色玻璃) - §e左 (A)
3. **S - 后** (橙色染色玻璃) - §6后 (S)
4. **D - 右** (红色染色玻璃) - §c右 (D)
5. 回到 **W**

### 物品显示

每个判定面对应不同颜色的染色玻璃：
- W: `WHITE_STAINED_GLASS`
- A: `YELLOW_STAINED_GLASS`
- S: `ORANGE_STAINED_GLASS`
- D: `RED_STAINED_GLASS`

## 时间导航

### 前进（向上滚动）

- 执行 `addTick(1)`
- 播放音效：`BLOCK_NOTE_BLOCK_HAT` (pitch 2.0)
- 显示标题：`§a>>> §7§o#拍数 §f位置§7/步长 §7(时间s) §a>>>`

### 后退（向下滚动）

- 执行 `addTick(-1)`
- 播放音效：`BLOCK_NOTE_BLOCK_HAT` (pitch 1.5)
- 显示标题：`§a<<< §7§o#拍数 §f位置§7/步长 §7(时间s) §a<<<`

### 显示信息

标题显示格式：
```
§a>>> §7§o#5 §f2§7/4 §7(4.50s) §a>>>
```

说明：
- `#5` - 第5拍
- `2/4` - 当前拍的第2个位置，步长为4（1/4拍）
- `4.50s` - 当前时间（秒）

## 注册流程

1. 在 `Main.onEnable()` 中注册 `EditorListener`：
```java
getServer().getPluginManager().registerEvents(new org.cubeRhythm.editor.EditorListener(), this);
```

2. 监听器自动检测玩家是否在编辑模式
3. 只有在编辑模式下才会响应滚轮事件

## 使用方法

1. 进入编辑模式：`/editor`
2. 创建或加载谱面：`/editor new <ID>` 或 `/editor load <ID>`
3. 使用鼠标滚轮：
   - 持有槽位0（音符类型）时滚动 → 切换音符类型
   - 持有槽位8（判定面）时滚动 → 切换判定面
   - 持有其他槽位时滚动 → 在时间轴上移动

## 技术细节

### 事件取消

```java
event.setCancelled(true);
```

取消事件以防止玩家实际切换物品栏槽位，保持编辑器界面稳定。

### 循环处理

处理 0↔8 的循环情况：
```java
if (previousSlot == 8 && newSlot == 0) {
    return true;  // 向上滚动
}
if (previousSlot == 0 && newSlot == 8) {
    return false;  // 向下滚动
}
```

### 状态同步

所有状态变化都保存在 `EditorSession` 中，确保：
- 状态持久化
- 多个系统可以访问当前状态
- 便于后续保存到文件
