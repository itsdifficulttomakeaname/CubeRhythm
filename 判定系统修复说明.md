### 3. DOUBLE 音符双击逻辑 ✅

**问题:** DOUBLE 音符只有变量定义，没有实现完整的双击检测逻辑

**修复位置:** `InputHandler.java:42-44, 147-189`

**修复内容:**
- 添加了 `firstClickJudgment` Map 用于存储第一次点击的判定结果
- 实现了完整的 `handleDoubleNote()` 方法：
  - **第一次点击**: 记录时间和判定结果，启动 0.5 秒定时器
  - **第二次点击**:
    - 检查是否在 0.5 秒内
    - 如果超时，判为 MISS
    - 如果在时间内，取两次判定中较差的结果
    - **如果任一为 JUST，则整体为 JUST**（符合规则）
  - **超时处理**: 0.5 秒后如果只有一次点击，自动判为 MISS
- 播放打击音效
- 显示判定文本并清理实体

**代码示例:**
```java
private void handleDoubleNote(NoteEntity noteEntity, JudgmentResult judgment, int timingOffset) {
    int currentCount = clickCount.getOrDefault(noteEntity, 0) + 1;
    clickCount.put(noteEntity, currentCount);

    if (currentCount == 1) {
        // 第一次点击
        firstClickTime.put(noteEntity, System.currentTimeMillis());
        firstClickJudgment.put(noteEntity, judgment);

        // 启动 0.5 秒定时器
        PlanetLib.getScheduler().runLater(() -> {
            if (clickCount.getOrDefault(noteEntity, 0) == 1 && !noteEntity.isHit()) {
                // 超时，判为 MISS
                processJudgment(noteEntity, JudgmentResult.MISS, 0);
                // 清理追踪数据...
            }
        }, 10L);
    } else if (currentCount == 2) {
        // 第二次点击
        long elapsed = System.currentTimeMillis() - firstClickTime.get(noteEntity);
        if (elapsed <= 500) {
            // 取两次判定中较差的结果
            JudgmentResult firstJudgment = firstClickJudgment.get(noteEntity);
            JudgmentResult finalJudgment;

            if (firstJudgment == JudgmentResult.MISS || judgment == JudgmentResult.MISS) {
                finalJudgment = JudgmentResult.MISS;
            } else if (firstJudgment == JudgmentResult.JUST || judgment == JudgmentResult.JUST) {
                finalJudgment = JudgmentResult.JUST;
            } else {
                finalJudgment = JudgmentResult.EXACT;
            }

            processJudgment(noteEntity, finalJudgment, timingOffset);
        } else {
            processJudgment(noteEntity, JudgmentResult.MISS, 0);
        }
        // 清理追踪数据...
    }
}
```

### 5. FLICK 音符旋转检测完善 ✅

**问题:** FLICK 音符逻辑简化，没有实际检查旋转角度

**修复位置:** `InputHandler.java:191-217`

**修复内容:**
- 完善了 `handleFlickNote()` 方法
- 点击时记录当前 yaw 角度
- 标记音符为已击中，防止重复判定
- 启动 0.5 秒定时器
- 定时器结束时：
  - 获取当前 yaw 角度
  - 使用 `ViewDirectionHelper.isRotatingInDirection()` 检查旋转方向和角度
  - 如果旋转正确（45°-135°），判为 EXACT
  - 如果旋转错误或未旋转，判为 MISS
- FLICK 音符无 JUST 判定（符合规则）
- 播放打击音效

**代码示例:**
```java
private void handleFlickNote(NoteEntity noteEntity, Player player, JudgmentResult judgment, int timingOffset) {
    if (judgment == JudgmentResult.MISS) {
        processJudgment(noteEntity, JudgmentResult.MISS, timingOffset);
        return;
    }

    // 记录点击时的 yaw
    float clickYaw = ViewDirectionHelper.getPlayerYaw(player);
    noteEntity.setHit(true);

    // 0.5 秒后检查旋转
    PlanetLib.getScheduler().runLater(() -> {
        if (noteEntity.isHit()) {
            float currentYaw = ViewDirectionHelper.getPlayerYaw(player);
            String targetDirection = noteEntity.getTurn();

            if (targetDirection != null && ViewDirectionHelper.isRotatingInDirection(
                player, clickYaw, currentYaw, targetDirection)) {
                // 旋转正确
                processJudgment(noteEntity, JudgmentResult.EXACT, 0);
            } else {
                // 旋转错误
                processJudgment(noteEntity, JudgmentResult.MISS, 0);
            }
        }
    }, 10L);
}
```

private NoteEntity findNoteEntityByInteraction(Interaction interaction) {
    for (NoteEntity entity : entityManager.getAllEntities()) {
        // 检查主 Interaction
        if (entity.getInteraction() != null &&
            entity.getInteraction().getUniqueId().equals(interaction.getUniqueId())) {
            return entity;
        }
        // 检查额外的 Interaction（DOUBLE 音符）

            }
        }
    }

## 📝 注意事项

### 1. 按键检测机制
HOLD 音符的按键检测使用了 100ms 窗口机制：
- 每次点击事件（左键或右键）会更新 `lastKeyPressTime`
- `isAnyKeyPressed()` 检查当前时间是否在最后一次按键的 100ms 内
- 这种机制可以有效检测玩家是否在持续按键

### 2. 定时器使用
- DOUBLE 音符的 0.5 秒超时：10 ticks
- FLICK 音符的 0.5 秒旋转检测：10 ticks
- 判定文本显示时间：20 ticks (1 秒)

### 3. 判定范围
- DRAG 和 HOLD 的自动判定触发范围：距离判定线 < 2 格
- 这个范围可以根据实际测试效果调整

### 4. 性能考虑
- 自动判定逻辑在每个 tick 执行，对性能影响较小
- 使用了 `isHit()` 标记防止重复判定
- 及时清理已判定的音符实体
