## 游戏概述

**Cube Rhythm** 是一款基于 Minecraft 的 3D 音乐节奏游戏，玩家站在一个立方体中心，四周（前后左右）会有音符从远处飞来，玩家需要在正确的时机点击音符以获得分数。

### 技术栈
- Minecraft 服务端
- Skript 插件脚本语言
- Display Entity (方块展示实体、文本展示实体)
- Interaction Entity (交互实体，用于检测点击)

### 5. 游戏开始阶段 (main.sk:225-291)

```
伪代码：
COMMAND /start:
    设置进度 = 50%

    // 对所有音符按时间排序

        复制到已排序数组

```

**自然语言描述：**
点击谱面后，系统会先对所有音符按时间排序（显示进度条），然后展示谱面信息，给玩家3秒准备时间，最后播放音乐并开始游戏。

### 6. 游戏进行阶段 (main.sk:293-652)

核心游戏循环（每 Tick 执行一次，1秒=20Ticks）：

```
伪代码：
EVERY_TICK:
    IF 计时器已启动:
        计时器 += 1
        更新进度条 = 计时器 / 歌曲总时长

        统计当前场景中的实体数量

        // 预生成音符（当音符即将进入可视范围）
        IF 场景实体数 < 100:
            FOR EACH 待生成音符:
                计算音符距离 = 流速 × 20 × (音符时间 - 当前时间) + 4

                IF 距离 < 50:  // 进入可视范围
                    根据音符类型生成对应实体:
                        - tap: 绘制Tap音符
                        - hold: 绘制Hold音符
                        - drag: 绘制Drag音符
                        - flick: 绘制Flick音符
                        - double: 绘制Double音符
                        - execution: 执行自定义脚本

                    从待生成列表移除

        // 移动所有音符
        FOR EACH 场景中的音符实体:
            根据判定面计算移动方向
            更新音符位置 -= 流速

            // 渲染音符缩放和透明度
            IF 音符在可视范围内:
                计算透明度 = 100 - (距离 - 4) × 4
                计算缩放 = 透明度 / 25
                更新显示效果

            // 判定逻辑
            IF 音符到达判定线:
                IF 自动演奏模式开启:
                    自动判定为Exact
                    删除音符
                ELSE:
                    根据音符类型处理:
                        - drag/hold: 检查是否按住
                        - flick: 检查是否转向

            // Miss判定
            IF 音符超出判定范围:
                删除音符
                触发Miss判定

        // 如果所有音符已消失
        IF 待生成音符数 = 0 AND 场景音符数 = 0:
            结束游戏
            进入结算
```

### 7. 玩家操作处理 (main.sk:55-159)

音符击打检测（通过多个事件监听）：

```
伪代码：
// 右键点击（主要检测方式）
ON_RIGHT_CLICK:
    IF 玩家是游戏角色 AND NOT 自动演奏:
        设置取消标记A = true
        获取准星指向的实体
        IF 实体是 tap/double 音符:
            执行 hitNote(实体)

// 射击事件（投掷雪球）
ON_SHOOT:
    IF 玩家是游戏角色 AND NOT 自动演奏:
        IF 没有被其他事件阻止:
            设置取消标记B = true
            获取准星指向的实体
            IF 实体是 tap/double 音符:
                执行 hitNote(实体)

// 左键点击
ON_LEFT_CLICK:
    IF 玩家是游戏角色 AND NOT 自动演奏:
        设置取消标记C = true
        获取准星指向的实体
        IF 实体是 tap/double 音符:
            执行 hitNote(实体)

// 伤害事件（用于处理点击碰撞箱）
ON_DAMAGE:
    IF 玩家是游戏角色 AND NOT 自动演奏:
        设置取消标记D = true
        获取准星指向或被击中的实体
        IF 实体是 tap/double 音符:
            执行 hitNote(实体)

// 挥手事件（用于修复双键bug）
ON_ARM_SWING:
    IF 玩家是游戏角色 AND NOT 自动演奏:
        IF 没有被其他事件阻止:
            获取准星指向的实体
            IF 实体是 tap/double 音符:
                执行 hitNote(实体)
```

**自然语言描述：**
系统通过监听多种点击事件（右键、左键、投掷、伤害、挥手）来捕捉玩家的击打操作。使用多重事件监听是为了确保在各种情况下都能准确检测到玩家的操作，同时使用取消标记机制防止重复触发。

## 音符系统

### 音符类型 (load.sk:69-96)

#### 1. Tap (点击音符)
```
特征：浅蓝色方块
判定：单次点击
生成函数：tap(时间, 判定面, X坐标, Y坐标, 是否发光, 标签)
```

#### 2. Hold (长按音符)
```
特征：白色长条方块
判定：需要按住直到音符到达判定线
生成函数：hold(时间, 判定面, X坐标, Y坐标, 是否发光, 标签)
```

#### 3. Drag (滑键音符)
```
特征：黄色方块
判定：准星瞄准后自动判定（玩家需要跟随音符移动视角）
生成函数：drag(时间, 判定面, X坐标, Y坐标, 是否发光, 标签)
```

#### 4. Flick (滑动音符)
```
特征：白色玻璃板 + 方向箭头
判定：点击后需要在0.5秒内转向指定方向
生成函数：flick(时间, 判定面, 转向方向, 是否发光, 标签)
```

#### 5. Double (双键音符)
```
特征：两个橙色方块 + 连接线
判定：需要同时点击两个音符（点击一个会自动消除另一个）
生成函数：double(时间, 判定面, X1, Y1, X2, Y2, 是否发光, 标签)
```

#### 6. Execution (执行音符)
```
特征：不可见
作用：在指定时间执行自定义脚本（用于场景动画、特效等）
生成函数：execution(时间, 脚本段)
```

### 音符渲染系统 (load.sk:98-485)

每种音符由两部分组成：
1. **Block Display Entity** (方块展示实体) - 视觉效果
2. **Interaction Entity** (交互实体) - 碰撞检测

```
伪代码：
FUNCTION drawTap(判定面, X, Y, Z, 发光, 标签):
    计算音符在3D空间的位置

    // 根据判定面旋转坐标
    根据判定面(w/a/s/d)转换坐标系

    // 生成方块展示实体
    在位置1生成方块展示:
        方块类型 = 浅蓝色混凝土
        缩放 = (1, 1, 0.5×流速)
        设置发光效果（如果启用）
        保存UUID用于后续删除

    // 生成交互实体（碰撞箱）
    在位置2生成交互实体:
        高度 = 2
        宽度 = 2
        元数据:
            音符类型 = "tap"
            判定面 = {判定面}
            关联UUID = {方块UUID}
            标签 = {标签}

    等待 2 Ticks
    设置方块亮度 = 最大
    设置平移动画使音符向判定线移动
```

## 判定系统

### 判定时间窗口 (judge.sk:195-225)

```
伪代码：
FUNCTION hitNote(音符实体):
    获取音符位置
    计算音符距离判定线的距离

    // 根据判定面计算时间差（单位：毫秒）
    时间差 = (距离 - 4) / 流速 × 50

    IF 绝对值(时间差) <= 80ms:
        触发 exact 判定(音符, 时间差)
        标记可删除
    ELSE IF 绝对值(时间差) <= 200ms:
        IF 音符标记为强制Exact:
            触发 exact 判定(音符, 时间差)
        ELSE:
            IF 时间差 > 0:
                触发 great 判定(音符, "early", 时间差)
            ELSE:
                触发 great 判定(音符, "late", 时间差)
        标记可删除

    IF 可删除:
        IF 音符类型是 double:
            同时删除关联的另一个音符
        删除音符
```

### 判定等级

#### Exact (完美判定)
```
时间窗口：±80ms
分数加成：+1.0
连击：+1
显示：蓝色 "Exact" 或偏移值
音效：打击音（如果开启）
```

#### Just (良好判定)
```
时间窗口：±81ms ~ ±200ms
分数加成：+0.7
连击：+1
显示：黄色 "Just" 或偏移值，并标注 Early/Late
音效：打击音（如果开启）
```

#### Miss (失误)
```
触发条件：音符超出判定范围未击打
分数加成：0
连击：清零
显示：红色 "Miss" 副标题
```

### 判定特效渲染 (judge.sk:25-188)

```
伪代码：
FUNCTION exact(音符实体, 时间偏移):
    连击数 += 1
    播放打击音效（如果开启）
    击中音符数 += 1
    Exact统计 += 1

    计算分数 = floor(1000000 / 总音符数 × 击中音符数)

    获取音符位置并转换到判定面坐标

    // 生成判定文本显示
    生成文本展示实体1（主文本）:
        IF 显示偏移开启:
            文本 = 时间偏移值（如 "+15" 或 "-8"）
        ELSE:
            文本 = "Exact"
        颜色 = 蓝色
        初始缩放 = 3

    生成文本展示实体2（背景光晕）:
        文本 = "█" (方块字符)
        颜色 = 蓝色半透明
        初始缩放 = 8

    // 动画循环（10帧）
    FOR 10次:
        降低透明度
        缩小主文本（收缩到1）
        扩大光晕（外扩效果）
        等待 1 Tick

    删除文本展示实体
```

### 特殊判定机制

#### Drag 音符判定 (judge.sk:7-23)
```
伪代码：
EVERY_TICK:
    IF NOT 自动演奏:
        获取玩家准星指向的实体

        IF 实体是 drag 音符:
            计算音符距判定线距离
            计算时间差

            IF 时间差在 ±200ms 内:
                将音符向上移动（视觉上"吸附"到准星）
                标记音符为"已击中"
```

**自然语言描述：**
Drag 音符不需要点击，只要玩家的准星在合适的时间指向音符，音符就会自动判定。这模拟了滑键的效果。

#### Flick 音符判定 (judge.sk:250-309)
```
伪代码：
FUNCTION checkIfTurned(音符实体):
    触发 exact 判定

    获取判定面和转向方向

    IF NOT 自动演奏:
        等待 0.5 秒

        获取玩家当前朝向yaw
        归一化yaw到 -180° ~ 180°

        // 根据判定面和转向方向检查玩家是否转向正确
        根据(判定面 × 转向方向)组合判断:
            IF 玩家yaw在正确的角度范围内:
                显示 "Pass"
            ELSE:
                触发 Miss
```

**自然语言描述：**
Flick 音符在点击后，玩家需要在0.5秒内将视角转向指定方向。系统会检查玩家的视角yaw值是否在正确的范围内（45度扇区）。

### 音符生成时机 (main.sk:293-417)

```
伪代码：
核心时序控制：
    当前时间(秒) = 计时器Ticks / 20
    音符距离 = 流速 × 20 × (音符时间 - 当前时间) + 4

音符生成触发条件：
    IF 场景中实体数 < 100 AND 音符距离 < 50:
        生成音符
        从待生成列表删除
```

**距离计算���明：**
- `音符时间 - 当前时间`：音符还有多久到达判定线（秒）
- `× 20`：转换为 Ticks
- `× 流速`：流速越快，提前生成距离越远
- `+ 4`：判定线距离玩家4格

示例：
```
流速 = 1.5
音符时间 = 10秒
当前时间 = 8秒
距离 = 1.5 × 20 × (10 - 8) + 4 = 64格

当距离 < 50 时才生成音符
即：1.5 × 20 × (t - 8) + 4 < 50
解得：t < 10.53秒

也就是说，这个音符会在当前时间达到约8.53秒时生成
```

### 1. 坐标系转换

游戏使用四个判定面（前w、左a、后s、右d），每个面有自己的局部坐标系。

```
伪代码：
FUNCTION convertCoordinates(判定面, localX, localY, localZ):
    IF 判定面 = "w" (前，面向+Z):
        worldX = localX
        worldY = localY
        worldZ = localZ
        旋转 = 0°

    ELSE IF 判定面 = "a" (左，面向-X):
        worldX = localZ
        worldY = localY
        worldZ = -localX

    ELSE IF 判定面 = "s" (后，面向-Z):

        worldY = localY
        worldZ = -localZ + 1

    ELSE IF 判定面 = "d" (右，面向+X):
        worldX = -localZ + 1
        worldY = localY
        worldZ = localX + 1
        旋转 = 270°

    返回 (worldX, worldY, worldZ, 旋转)
```

### 3. 缓动函数系统 (api.sk:248-350)

用于实现平滑的非线性动画（如结算界面UI飞入）。

```
伪代码：
FUNCTION easingMotion(实体, 起点, 终点, X缓动, Y缓动, Z缓动, 旋转缓动, 持续帧数):
    提取起点和终点的XYZ坐标和旋转

    FOR 持续帧数 次:
        进度百分比 = 当前帧 / 总帧数

        X进度 = calcEasingValue(进度百分比, X缓动类型)
        Y进度 = calcEasingValue(进度百分比, Y缓动类型)
        Z进度 = calcEasingValue(进度百分比, Z缓动类型)
        旋转进度 = calcEasingValue(进度百分比, 旋转缓动类型)

        当前X = 起点X + (终点X - 起点X) × X进度
        当前Y = 起点Y + (终点Y - 起点Y) × Y进度
        当前Z = 起点Z + (终点Z - 起点Z) × Z进度
        当前旋转 = 起点旋转 + (终点旋转 - 起点旋转) × 旋转进度

        传送实体到(当前X, 当前Y, 当前Z, 当前旋转)
        等待 1 Tick

FUNCTION calcEasingValue(x, 类型):
    // 线性
    IF 类型 = "linear" 或 "1":
        返回 x

    // 正弦缓动
    IF 类型 = "easeInSine" 或 "sina":
        返回 1 - cos((x × π) / 2)

    IF 类型 = "easeOutSine" 或 "sinb":
        返回 sin((x × π) / 2)

    // 圆形缓动（常用于结算UI）
    IF 类型 = "easeOutCirc" 或 "cirb":
        返回 sqrt(1 - (x - 1)²)

    // 二次缓动
    IF 类型 = "easeInQuad" 或 "2a":
        返回 x²

    IF 类型 = "easeOutQuad" 或 "2b":
        返回 1 - (1 - x)²

    ... (还有更多缓动函数)
```

**常用缓动类型：**
- `"1"` / `"linear"`: 匀速
- `"cirb"` / `"easeOutCirc"`: 快速启动，平滑减速（结算UI常用）
- `"sina"` / `"easeInSine"`: 慢速启动，加速
- `"sinb"` / `"easeOutSine"`: 快速启动，减速

### 4. 分数计算

```
伪代码：
总分 = 1,000,000

单个音符分数 = 1,000,000 / 总音符数

Exact判定：击中音符数 += 1.0
Just判定：击中音符数 += 0.7
Miss判定：击中音符数 += 0

当前分数 = floor(1,000,000 / 总音符数 × 击中音符数)

Hold音符特殊加分：
    每个Hold音符完美通过 +1 分（不受音符数影响）
    最终分数 = 基础分数 + Hold加分
```

**Perfect (SSS+) 条件：**
```
所有音符都Exact + 所有Hold都完美 = 1,000,000 分
```

**示例计算：**
```
谱面有 100 个音符（其中5个Hold）

完美游玩：
- 100 × Exact = 100 × 1.0 = 100
- 分数 = floor(1000000 / 100 × 100) + 5 = 1,000,005

实际显示为 1,000,000 + 5

部分失误：
- 90 × Exact + 8 × Just + 2 × Miss = 90 + 5.6 + 0 = 95.6
- 分数 = floor(1000000 / 100 × 95.6) + 3 = 956,003
```

```
临时游戏数据:
    {actor} = 当前游玩的玩家
    {playing} = 当前游玩的谱面ID
    {timer} = 游戏计时器(Ticks)
    {combo} = 当前连击数
    {maxNote} = 总音符数
    {score} = 当前分数
    {hitNotes} = 已击中音符数（带小数）
    {statistics::exact} = Exact数量
    {statistics::just} = Just数量
    {statistics::miss} = Miss数量
    {statistics::exactHold} = Hold完美数量
    {loadedNotes::*} = 已加载的音符数据
```

### 6. 音频同步

```
音频播放时机：
    播放预览 = 选择谱面后 +2秒
    停止预览 = 开始加载后
    播放正式 = 所有准备完成后 +3秒

音符时间调整：
    原始时间 = 谱面定义的时间
    调整后时间 = 原始时间 + 3秒 + 玩家偏移 + 谱面偏移

    3秒：预留的准备时间
    玩家偏移：补偿网络延迟/客户端差异
    谱面偏移：补偿音频文件起始延迟

判定时间计算：
    音符位置 → 距离 → 时间差(毫秒)

    距离 = 音符Z坐标 - 判定线坐标
    时间(秒) = 距离 / 流速 / 20
    时间(毫秒) = 时间(秒) × 1000 / 2.5  (实际代码: × 50)
```
