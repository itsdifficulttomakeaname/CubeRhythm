# 编辑器音符渲染系统说明

## 功能概述

实现了编辑器中的音符可视化渲染系统，当加载谱面或在时间轴上导航时，会自动显示当前位置附近的音符。

## 核心类

### EditorNoteRenderer

编辑器专用的音符渲染器，负责在编辑模式下显示音符。

**主要方法**：

```java
// 渲染当前时间范围内的可见音符
public static void renderVisibleNotes(EditorSession session, Player player)

// 清理所有音符显示实体
public static void clearNoteDisplays(EditorSession session)
```

## 渲染规则

### 时间范围

- **渲染范围**：当前时间前后 5 秒内的音符
- **常量定义**：`RENDER_TIME_RANGE = 5.0`

### 距离计算

```java
double distance = speed * 20 * timeDiff + 4
```

- `timeDiff`：音符时间 - 当前时间
- `speed`：流速（从 EditorSession 获取）
- 距离 4 是判定线位置

### 音符类型渲染

| 音符类型 | 材质 | 特殊处理 |
|---------|------|---------|
| TAP | LIGHT_BLUE_CONCRETE | 1x1x1 方块 |
| HOLD | WHITE_CONCRETE | 1x1x3 拉长方块 |
| DRAG | YELLOW_CONCRETE | 1x1x1 方块 |
| FLICK | MAGENTA_CONCRETE | 5x5x1 大方块，位于中心 |
| DOUBLE | ORANGE_CONCRETE | 两个 1x1x1 方块 |
| EXECUTION | BARRIER | 不渲染 |

### 坐标转换

使用 `CoordinateSystem.transformCoordinates()` 将本地坐标转换为世界坐标：

- **W/S 面**：不需要额外偏移
- **A/D 面**：X 坐标右移 1 格

### 发光效果

如果音符的 `glowing` 属性为 `true`，会启用 BlockDisplay 的发光效果。

## 触发时机

音符渲染会在以下情况下自动触发：

### 1. 加载谱面时

```java
// EditorFileUtil.loadChartToEditor()
EditorNoteRenderer.renderVisibleNotes(session, session.getPlayer());
```

### 2. 时间轴导航时

```java
// EditorListener.handleBeatNavigation()
session.addTick(amount);
EditorNoteRenderer.renderVisibleNotes(session, player);
```

### 3. 跳转到指定拍时

```java
// BeatCommand.onCommand()
session.setTick((beat - 1) * session.getStepLength());
EditorNoteRenderer.renderVisibleNotes(session, player);
```

### 4. 修改步长时

```java
// StepCommand.onCommand()
session.setStepLength(stepLength);
EditorNoteRenderer.renderVisibleNotes(session, player);
```

## 清理机制

音符显示实体会在以下情况下被清理：

### 1. 退出编辑模式时

```java
// EditorManager.endSession()
EditorNoteRenderer.clearNoteDisplays(session);
```

### 2. 插件卸载时

```java
// EditorManager.cleanup()
for (EditorSession session : sessions.values()) {
    EditorNoteRenderer.clearNoteDisplays(session);
}
```

### 3. 重新渲染时

```java
// EditorNoteRenderer.renderVisibleNotes()
clearNoteDisplays(session);  // 先清理旧的
// 然后渲染新的
```

## 实现细节

### 单个音符渲染

```java
private static void renderSingleNote(EditorNote note, World world, Material material, double distance) {
    // 1. 计算面偏移
    double faceOffsetX = (note.getFace() == Face.A || note.getFace() == Face.D) ? 1.0 : 0.0;

    // 2. 转换坐标
    Location location = CoordinateSystem.transformCoordinates(...);

    // 3. 创建 BlockDisplay
    BlockDisplay display = DisplayEntityFactory.createBlockDisplay(...);

    // 4. 设置发光
    if (note.isGlowing()) {
        display.setGlowing(true);
    }

    // 5. 保存到音符
    note.setBlockDisplay(display);
}
```

### DOUBLE 音符渲染

DOUBLE 音符需要渲染两个位置：

```java
// 第一个位置
BlockDisplay display1 = DisplayEntityFactory.createBlockDisplay(...);
note.setBlockDisplay(display1);

// 第二个位置
BlockDisplay display2 = DisplayEntityFactory.createBlockDisplay(...);
note.getAdditionalDisplays().add(display2);
```

### FLICK 音符渲染

FLICK 音符在判定面中心，尺寸为 5x5x1：

```java
Location location = CoordinateSystem.transformCoordinates(
    world, note.getFace(),
    0, 0,  // 中心位置
    distance, ...
);

BlockDisplay display = DisplayEntityFactory.createBlockDisplay(
    world, location, material,
    5.0f, 5.0f, 1.0f,  // 5x5x1 大小
    100
);
```

## 性能优化

### 渲染范围限制

只渲染时间范围内的音符（前后 5 秒），避免渲染过多实体：

```java
if (Math.abs(timeDiff) <= RENDER_TIME_RANGE) {
    renderNote(note, world, session.getSpeed(), timeDiff);
}
```

### 实体复用

每次渲染前先清理旧实体，避免实体累积：

```java
clearNoteDisplays(session);  // 清理旧的
// 渲染新的
```

### EditorNote 清理

EditorNote 的 `cleanup()` 方法会清理所有关联的显示实体：

```java
public void cleanup() {
    if (blockDisplay != null && !blockDisplay.isDead()) {
        blockDisplay.remove();
    }
    for (BlockDisplay display : additionalDisplays) {
        if (display != null && !display.isDead()) {
            display.remove();
        }
    }
    additionalDisplays.clear();
}
```

## 使用示例

### 加载谱面并查看音符

```
1. /editor                    # 进入编辑模式
2. /editor load my_chart      # 加载谱面
   → 自动渲染当前位置（第0拍）附近的音符
3. 滚动鼠标轮前进/后退
   → 每次移动都会更新音符显示
4. /b 10                      # 跳转到第10拍
   → 自动渲染第10拍附近的音符
```

### 调整步长并查看效果

```
1. /step 4                    # 设置步长为1/4拍
   → 重新渲染音符
2. /step 8                    # 改为1/8拍
   → 重新渲染音符
```

## 视觉效果

### 音符颜色

- **TAP**：浅蓝色
- **HOLD**：白色（拉长）
- **DRAG**：黄色
- **FLICK**：品红色（大方块）
- **DOUBLE**：橙色（两个方块）

### 发光效果

带有 `glowing: true` 属性的音符会发光，更容易识别。

### 距离效果

- 未来的音符：距离 > 4，在判定线前方
- 当前音符：距离 ≈ 4，在判定线上
- 过去的音符：距离 < 4，在判定线后方

## 与游戏渲染的区别

| 特性 | 游戏渲染 | 编辑器渲染 |
|------|---------|-----------|
| 渲染范围 | 50 格距离 | 前后 5 秒 |
| 动态更新 | 每 tick 更新 | 导航时更新 |
| 缩放效果 | 有渐变缩放 | 固定大小 |
| 光标显示 | 有 TextDisplay 光标 | 无光标 |
| 碰撞箱 | 有 Interaction 实体 | 无碰撞箱 |
| 性能要求 | 高（实时游戏） | 低（静态编辑） |

## 故障排除

### 音符不显示

1. 检查是否加载了谱面
2. 检查当前时间范围内是否有音符
3. 检查玩家是否在正确的世界

### 音符位置错误

1. 检查坐标系统配置
2. 检查面偏移计算
3. 检查 CoordinateSystem 实现

### 性能问题

1. 减小 `RENDER_TIME_RANGE` 值
2. 优化音符数量
3. 检查是否有实体泄漏

## 总结

编辑器音符渲染系统提供了直观的可视化反馈，让谱面制作者能够实时看到音符的位置和类型，大大提升了编辑体验。系统设计简洁高效，与现有的编辑器架构无缝集成。
