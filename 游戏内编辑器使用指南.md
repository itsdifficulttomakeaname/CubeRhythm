# 游戏内铺面编辑器使用指南

## 概述

CubeRhythm 现已实现完整的游戏内铺面编辑器，支持实时文件保存、谱面加载等功能。

## 主要功能

### 1. 编辑器模式管理

- `/editor` - 进入/退出编辑模式
- `/editor help` - 查看完整帮助信息

### 2. 文件管理

- `/editor new <谱面ID>` - 创建新谱面
  - 自动检查ID是否重复
  - 立即创建JSON文件
  - 后续修改自动保存

- `/editor load <谱面ID>` - 加载现有谱面到编辑器
  - 支持加载所有音符类型
  - 自动读取BPM、PreTime等元数据

- `/editor save` - 手动保存谱面
  - 通常不需要手动保存，系统会自动保存

### 3. 参数设置

- `/editor bpm <数值>` - 设置BPM
  - 影响HOLD音符长度计算
  - 自动保存到文件

- `/editor pretime <秒数>` - 设置第一拍前的时间
  - 用于给玩家准备时间

- `/editor speed <倍速>` - 设置流速
  - 影响音符移动速度和HOLD音符长度

### 4. 时间轴操作

- `/step <整数>` - 设置步长
  - 例如 `/step 4` 表示1/4拍
  - 例如 `/step 8` 表示1/8拍

- `/b <拍数>` - 跳转到指定拍
  - 例如 `/b 10` 跳转到第10拍

- **鼠标滚轮** - 前进/后退一格
  - 向上滚动：前进一格
  - 向下滚动：后退一格

### 5. 音符操作

#### 工具栏（自动给予）

- **Slot 0** - 音符类型选择器
  - 按 `1` 键循环切换
  - TAP → DOUBLE → DRAG → HOLD → FLICK_L → FLICK_R → TAP

- **Slot 4** - 魔杖（主要工具）
  - 右键：放置音符
  - 左键：删除音符
  - Shift+右键：对齐到网格

- **Slot 8** - 判定面选择器
  - 按 `9` 键循环切换
  - W（前）→ A（左）→ S（后）→ D（右）→ W

- **F键** - 切换发光效果

## HOLD音符长度修复

HOLD音符的长度现在由BPM和流速决定，计算公式为：

```
长度 = (60 / BPM) × 流速
```

这确保了：
- 一拍的HOLD音符能与下一拍的音符首尾相接
- 连续的HOLD音符看起来更连贯
- 不同BPM下HOLD音符长度自动适配

## 实时文件保存

编辑器采用实时保存机制：

1. **创建谱面时**：立即创建JSON文件
2. **修改参数时**：自动保存（BPM、PreTime等）
3. **放置/删除音符时**：自动保存
4. **退出编辑器时**：自动保存

文件保存位置：`plugins/CubeRhythm/<谱面ID>.json`

## 使用流程示例

### 创建新谱面

```
1. /editor                    # 进入编辑模式
2. /editor new my_chart       # 创建新谱面
3. /editor bpm 140            # 设置BPM
4. /editor speed 1.5          # 设置流速
5. /step 4                    # 设置步长为1/4拍
6. [使用魔杖放置音符]
7. /editor                    # 退出（自动保存）
```

### 编辑现有谱面

```
1. /editor                    # 进入编辑模式
2. /editor load my_chart      # 加载谱面
3. /b 10                      # 跳转到第10拍
4. [修改音符]
5. /editor save               # 手动保存（可选）
6. /editor                    # 退出
```

## 注意事项

1. **谱面ID唯一性**：创建新谱面时会自动检查ID是否已存在
2. **自动保存**：大部分操作会自动保存，无需手动保存
3. **BPM影响**：修改BPM会影响HOLD音符的长度显示
4. **流速影响**：流速影响所有音符的移动速度和HOLD音符长度

## 技术实现

### 核心类

- `EditorSession` - 管理单个玩家的编辑状态
- `EditorManager` - 管理所有编辑器会话
- `EditorNote` - 编辑器中的音符数据
- `EditorFileUtil` - 文件保存/加载工具
- `EditorCommand` - 主命令处理器
- `StepCommand` - 步长命令
- `BeatCommand` - 跳转命令

### HOLD音符长度计算

在 `NoteRenderer` 中：

```java
float scaleZ = entity.getType() == NoteType.HOLD
    ? (float) ((60.0 / bpm) * speed)  // 一拍的距离
    : (float) (0.5 * speed);
```

这确保HOLD音符的长度精确对应一拍的时间。
